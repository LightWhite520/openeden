# .env 自动加载设计

## 目标

让 OpenEden 的 CLI 和 Ktor server 在启动时自动读取项目根目录的 `.env`，用户无需先手动执行 PowerShell 环境变量导入命令。

## 行为

- 进程环境变量优先于 `.env` 中的同名变量。
- `.env` 不存在时继续使用系统环境变量和代码默认值，不阻止启动。
- 支持空行、以 `#` 开头的注释、首尾空白，以及单引号或双引号包裹的值。
- 不支持变量展开、多行值或复杂 shell 语法。
- `.env` 仅作为本地配置文件，继续被 Git 忽略；`.env.example` 可提交。

## 架构

在 JVM 公共配置层提供一个无副作用的 dotenv 解析器。启动入口先读取 `.env`，再与 `System.getenv()` 合并，合并时保留进程环境变量。CLI 配置和 Ktor runtime 统一使用合并后的 map，避免两套配置读取规则。

## 错误处理

无法读取 `.env` 时忽略该文件并继续启动；格式不完整的行跳过，不把配置文件问题升级为阻塞启动错误。真正需要的配置，例如 OpenAI API Key，仍由现有配置校验负责报错。

## 测试

- 解析注释、空行和引号值。
- `.env` 值能补充缺失的系统环境变量。
- 系统环境变量覆盖 `.env` 值。
- 缺少 `.env` 时保留现有默认行为。
